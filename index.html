履程：面向香港与内地，把履历对齐真实岗位

地区就业市场痛点

香港

  • 国际金融与多元产业并存，简历需要经得住行业逻辑的审视，而非简单信息堆砌。
  • 筛选与匹配成本高，求职者难以及时看清岗位叙事与自身材料的差距。

中国内地

  • 环境高度竞争，雇主极看重垂直实习与对口经历，履历与 JD 之间常见「匹配断层」。
  • 职业规划近乎刚需，传统高价咨询却将多数普通学生挡在门外，信息与服务仍不对称。

向下可展开产品闭环、技术选型与社会效益；能力由方舟大模型 + RAG + 流式交付支撑，把对齐 JD、补经历与可执行规划收成一条你能跟完的链路。

产品闭环、技术选型与社会效益

把散落在各处的经历，收成模型能读懂、敢下判断的一份叙事——再交回你手里签字。

产品闭环 六步 · 录入 → 规划
  1. 录入 — 简历与多份 JD 结构化入库，解析绑定会话，后续分析、改写、追问共用同一份可对齐的上下文。
  2. 分析 — 简历 × 多岗位匹配推理；长报告用 SSE 渐进送达，少白屏、多「看得见在推」。
  3. 改写 — 按教育 / 实习 / 项目等模块分段生成；口述经历经 ASR 再进大模型整理与改写，多模态进同一条管线。
  4. 补信息 — 模型驱动追问与选项，缺口补全后写回会话，再喂回生成链路。
  5. 核对 — 原文与改写并排、差异高亮，确认后再采纳，闭环可审计。
  6. 规划 — 模型归纳投递时间线与行动建议，长内容支持屏内滚动读完，从材料落到「下一步干嘛」。
技术选型 模型 · RAG · 工程栈
  • 火山引擎方舟（豆包）大模型 — OpenAI 兼容 API（北京区域端点），统一客户端与模型版本切换。
  • 检索增强（RAG） — 岗位描述与简历结构化片段检索相关上下文，再交由大模型做分析、追问与改写；索引形态随迭代对齐。
  • 多场景 prompt 与链式编排 — 分析、改写、追问、规划、语音整理等分场景模板与调用链（可配合 LangChain 组织复杂逻辑）。
  • SSE 流式分析 — 长上下文与多 JD 下服务端推送分段呈现，避免一次等待大段 JSON 的断裂感。
  • ASR + LLM — 火山语音识别与大模型总结 / 追问衔接；前端可选用 WebSocket 流式转写，口述快速进入结构化流程。

工程底座：React、TypeScript、Vite、Tailwind；FastAPI；Supabase（PostgreSQL）；前端 Vercel。交付与扩展的底子，差异化仍在模型与数据链。

社会效益 普惠 · 匹配 · 赋权

简历与 JD 错位、招聘初筛成本高，叠加高价职业规划，常把普通求职者挡在机会之外。履程用 AI 把深度对齐岗位、挖掘既有优势与实习路径建议做成可反复使用的辅助，压低门槛，让缺少资源背景的人也能持续获得像样的支持。

人与岗匹配更准，盲投与误筛略减，社会层面的人力资源配置会更健康；用技术降低信息与服务的门槛，也是在放大普通人的可见度与选择权。
